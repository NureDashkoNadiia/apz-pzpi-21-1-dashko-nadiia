1.1. Background (Передумови) 


Сільське господарство, зокрема вирощування овочів, фруктів та квітів у теплицях, є важливою галуззю, яка забезпечує населення свіжими, якісними продуктами та рослинами протягом усього року. Робота з теплицями кропітка та потребує особливої уваги, а невідповідність умов для росту рослин може спричинити їх дефіцит на ринку. Є проблема ризику втрат врожаю через людський фактор, коли помилки або недогляд персоналу можуть спричинити пошкодження або загибель рослин. Крім того, ручне виконання операцій, таких як полив чи внесення добрив, часто призводить до нераціонального використання ресурсів, зокрема води, енергії та добрив. Збір та аналіз даних про процеси вирощування для оптимізації технологій також є складним завданням при ручному веденні справ. Нарешті, догляд за рослинами потребує значної кількості працівників, що збільшує витрати на персонал та обмежує можливості масштабування виробництва. 
Автоматизована система для вирощування рослин може розв’язати всі ці проблеми, беручи на себе контроль умов, операції догляду та збір даних, тим самим зводячи до мінімуму участь людини, що дозволяє оптимізувати процеси вирощування, підвищити врожайність, заощадити ресурси та знизити витрати на персонал.

1.2. Business Opportunity (Бізнес-можливості) 

Розглянемо аналоги системі AutoFarm.  
Priva – одна з провідних компаній у галузі автоматизації та клімат-контролю для теплиць. Priva пропонує комплексні рішення для управління мікрокліматом, поливом, енергоспоживанням та іншими процесами в теплицях. Вони мають безліч готових продуктів, розглянемо найпопулярніші: 
− Priva Connext – центральна система управління для контролю над усіма процесами в теплицях. Вона інтегрує різні компоненти, такі як датчики, виконавчі механізми, системи зрошення тощо; 
− Priva FS Performance – спеціалізована система для оптимізації живлення рослин та поливу в теплицях; 
− Priva TopControl – програмне забезпечення для контролю різних аспектів вирощування, включаючи клімат, енергоспоживання, зрошення тощо. 
Системи від цієї компанії використовуються у великих комерційних теплицях по всьому світу для вирощування овочів, фруктів, квітів тощо. Крім обладнання та програмного забезпечення, Priva також пропонує послуги з проєктування, монтажу та обслуговування систем автоматизації для теплиць. 
Перейдемо до наступного аналога. Argus Control Systems – це канадська компанія, що спеціалізується на розробці систем автоматизації та клімат-контролю для теплиць. Argus пропонує свій флагманський продукт – Argus Greenhouse Control System, який є комплексним рішенням для моніторингу та управління мікрокліматом у теплицях. Система дозволяє контролювати температуру, вологість, рівень CO2, освітлення, системи зрошення, затінення та інші ключові параметри. Цей програмний продукт надає можливість віддаленого моніторингу та управління через вебінтерфейс або мобільний додаток. Крім того, на відміну від продуктів Priva, завдяки Argus Greenhouse Control System можна проводити аналітику даних для оптимізації процесів вирощування. 
Основною діяльністю Argus Control Systems є співпраця з дослідницькими установами та надання спеціалізованих рішень для наукових проєктів у галузі садівництва та сільського господарства. 
У порівнянні з Priva та Argus Control Systems, AutoFarm не займається створенням самого обладнання (датчиків, виконавчих механізмів тощо) для теплиць. AutoFarm –  це саме система управління, яка інтегрується з обладнанням, що вже є у клієнта. Тобто, клієнту надається можливість закупити тільки необхідні йому компоненти для теплиці та самому включити їх у цю систему. 
Головною перевагою AutoFarm над аналогами є економія коштів клієнта. Оскільки необхідність купувати апаратні компоненти разом з програмним забезпеченням відсутня, така ситуація може знизити загальну вартість рішення для клієнтів, які вже мають деяке обладнання. Спираючись на це, на відміну від Priva та Argus Control Systems, AutoFarm підходить як для великих теплиць, так і для маленьких домашніх ферм. Також перевагою є те, що клієнти можуть обирати обладнання різних виробників, не прив'язуючись до рішень однієї компанії. Це забезпечує більшу гнучкість та свободу вибору. Наступною перевагою є простота інтеграції. Замість того, щоб узгоджувати власне обладнання з програмним забезпеченням, система розробляється для інтеграції з широким спектром наявних пристроїв та датчиків на ринку.  
Монетизація проєкту відбувається через продаж ліцензій на використання програмного забезпечення (одноразова або періодична підписка) та послуг з налаштування, інтеграції та технічної підтримки. 
Цільовим ринком програмного продукту є власники теплиць, домашніх ферм та фермерських господарств різних розмірів, зосереджених на вирощуванні овочів, фруктів, квітів тощо. Ринок сільського господарства зростає у зв'язку з підвищенням попиту на свіжу та екологічну продукцію, тож пропозиція щодо автоматизації вирощування рослин має хороші перспективи для генерування прибутку внаслідок високого попиту та можливості масштабування.

2.1. Vision Statement (Окреслення концепції) 

AutoFarm є ідеальним рішенням для досягнення майже повної автоматизації роботи теплиць та домашніх мікроферм. Тобто незважаючи на автоматизацію більшості процесів, за теплицею все одно необхідно наглядати та виконувати певні ручні операції. У випадку нестачі або виходу з ладу датчиків чи виконавчих механізмів, полив або внесення добрив доведеться здійснювати вручну. Також потрібен регулярний догляд за рослинами, видалення бур'янів, боротьба зі шкідниками тощо.  
Основною перевагою цієї програмної системи є універсальність, оскільки вона дозволяє управляти різними видами датчиків та виконавчих механізмів (полив, освітлення тощо) незалежно від їх виробника чи системи управління, якою вони користуються. За допомогою цього продукту можна легко інтегрувати наявне обладнання для теплиць в єдину централізовану систему управління, що покращує його контроль. 
Особливо важливою є можливість моніторингу та аналізу даних з датчиків, які клієнт може приєднати до системи. Ця функція дозволяє отримувати вичерпну інформацію про параметри мікроклімату та умови вирощування, серед них: температура повітря, вологість повітря, рівень вуглекислого газу (CO2), освітленість, зволоженість ґрунту та інші. 
Крім того, система AutoFarm надає можливість налаштовувати параметри виконавчих механізмів згідно з конкретними потребами та вимогами, які встановлює сам клієнт. Це означає, що ви можете легко адаптувати обладнання до різних завдань і оптимізувати робочий процес для досягнення максимальної продуктивності. Таким чином, цей продукт стає надійним і ефективним помічником у підтримці сучасних теплиць та домашніх мікроферм.

4.3. Operating Environment (Робоче середовище) 
 
Front-end частина системи буде розроблена з використанням мови JavaScript, та додатково в ній буде використовуватися потужний фреймворк React. Мова розмітки HTML та мова стилів сторінок CSS також будуть активно використовуватися в цій частині системи. Паралельно з цим, до мови CSS буде застосовано розширення – LESS, щоб забезпечити більшу гнучкість та продуктивність. Розробка front-end частини буде проводитися у середовищі Visual Studio Code. 
Back-end системи буде розроблений мовою JavaScript в програмному середовищі Node.js з використанням вебфреймворку Express.js. Розробка back-end частини буде проводитися у середовищі Visual Studio Code. Система управління базами даних MySQL буде використовуватися для ефективного зберігання та обробки даних. Для роботи з базою даних буде використовуватися інструмент MySQL Workbench. 
Мобільний застосунок, який інтегрується в систему, буде розроблений на мові Swift з використанням розширеного функціонала фреймворку UIKit. Для розробки мобільних додатків на платформі iOS буде використовуватися середовище розробки Xcode. 
Частина, що відноситься до Internet of Things (IoT), буде виконана мовою C++ за допомогою середовища Arduino IDE.  Бібліотека DHT дозволить працювати з популярними датчиками температури та вологості повітря DHT11 та DHT22. Для налагодження бездротового підключення пристрою та передачі зібраних даних на хмарні платформи буде використовуватися WiFi.h, що забезпечує роботу з Wi-Fi модулями на основі чіпів ESP8266 або ESP32. Нарешті, для організації обміну даними буде залучена бібліотека PubSubClient.  
Система має високі вимоги до доступності, надійності, продуктивності та цілісності. Вона повинна бути доступною цілодобово і, в разі виникнення збоїв в роботі, передбачати створення резервних компонентів для безперебійної роботи. Для забезпечення швидкої реакції на запити користувачів, система має бути швидкою та ефективною, оптимізованою з урахуванням великої кількості користувачів і вимог до продуктивності.
